# Financial-Risk-Analysis-Blackrock--Python
This project analyzes customer financial transactions at BlackRock to uncover behavior patterns and risks. Using Python, data is cleaned and explored through EDA to identify trends, unusual activities, and insights that support better decision-making and improve customer engagement.
### Objective:
To Build a complete Customer Financial Behavior and Risk Analysis Report using Python for data handling. This report should clean and consolidate Blackrock’ customer account data, uncover transaction insights, and help decision-makers align customer strategies with usage trends and financial risk indicators.
### Tools Used:
Jupyter Notebook, python Libraries(NumPy, Pandas, Matplotlib, Seaborn)
### Dataset:
This dataset provides a transactional summary of Blackrock’ customer accounts, including account numbers, transaction dates, transaction types (credit/debit), transaction amounts, account types, and available balances.
### Tasks Performed:
Data Cleaning, Data preprocessing.
Transactional Analysis, Identifyied financial risks, Data visualization using libraries
Hypothesis testing
### Key Insights
Customer segment strongly influences transaction volume and frequency.
Majority of users fall into medium engagement, driving overall activity.
Customers are more inclined toward money movement/spending rather than saving.
Account type defines purpose-driven behavior.
### Reasons for financial risk?
Outliers-Very large transaction amounts detected which may risk for possible fraud or unusual activity.
Irregular Transaction gap that is more than 60 days risks customers inactivity and churning of customers.
High volatility that is large fluctuation in transaction amounts risks unstable financial behavior.
Analysis shows high withdrawal patterns which risks liquidity issues or risky spending.
### Recommendations
Personalized offers for medium-activity customers.
Onboarding incentives for new users
Premium services for high-value customers.
Additionally, implementing monitoring systems for high-value transactions, inactive accounts, and volatile behavior can help mitigate financial risks and improve overall customer management.”
